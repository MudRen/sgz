Emote 新规则

通常的 action 和 emote 都是采用 m_messages.c 这个 module 来处理
动作对象的名称、代词以及称呼的。经过一些修改，新的规则如下：

1. 所有的对象代换使用 $N[<digit>][<digit>][<symbol>] 的形式。其中
   $N 之后可以附加一个或两个数字，还可以附加一个符号。$N 也可以被
   写作 $n.

2. 第一个数字表示第几个对象，比如 emote hit 有一条：
        LIV -> $N0一拳打在$N1的肚皮上。
   其中 $N0 表示施动者，$N1 表是第一个对象。

3. 附加的 symbol 可以表示对所指对象使用不同的名称。其规则如下：

        n     表示对象的名字，也就是通常的 SHORT。但是如果这条 message
              是给你自己看的，那都会换成“你”。

	m     表示对象的名字，也就是通常的 SHORT。但不会被换成“你”。

        o     也表示对象的名字，但是如果此对象在同一条 message 里
              出现过了，则表示对象的的第三人称代词，如他，她或它。
              而且如果此对象是施动者($N0)，则会表示“某某自己”。
              比如 hit LIV -> $N0一拳打在$N1o的肚皮上。
                如果 stefan 输入 hit fire 的话...
              stefan 会看到：你一拳打在烈火的肚皮上。
                fire 会看到：斯坦福一拳打在你的肚皮上。
                 别人会看到：斯坦福一拳打在烈火的肚皮上。
              而如果 stefan 输入 hit stefan 的话...
              stefan 会看到：你一拳打在你自己的肚皮上。
                 别人会看到：斯坦福一拳打在他自己的肚皮上。  

        p     表示对象的第三人称代词。(pronoun)
        
        c     表示此对象相对于施动者($N0)的亲昵称呼。(close)
	      比如：
                 greet LIV -> $N0对$N1作揖道：$N1c一向可好？
              如果在 $N 和 c 之间有两个 digit，那么第一个 digit 表
              示所参照的对象，第二个 digit 表示此对象，比如：
                 intro LIV to LIV -> $N0对$N2说：$N1是你的$N21c。

        a     表示对象的尊称。(admirable)

        h     表示对象的谦称。(humble)

        r     表示对象的蔑称。(rude)

4. $O 或 $o 用来表示 LIV 之外的东西，比如 OBJ, STR 等。其格式与 $N
   大同小异。但是后面附加 symbol 就不起作用了。

5. 一些缩写：
      $N = $N0n      $n = $N0o
      $T = $N1n      $t = $N1o
      $M = $N0m      $m = $N1m
      $C = $N01c     $c = $N10c
      $S = $N0a	     $s = $N0h
      $R = $N1a      $r = $N1r
   其中 $C $c $S $s $R $r 的定义略有出入，正确的定义请参照 oldemoterule。
   保留这些缩写是为了和旧的规则相兼容，但还是尽量请用新规则。

6. 相对于英文 Lima 中 message rule 的变动有：
       取消了主格宾格的变化，取消了动词单复数的变化，取消了所有格，
       增加了几种不同的称呼。
